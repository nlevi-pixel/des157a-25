Form Design Principles

When beginning to practice designing forms, it is important to revert back to basic design principles while adhering to current UX/UI patterns. For instance, a core design principle is to create contrast, similarly present web design patterns require contrast for usability and accessibility. Designing forms is uniquely important, as it can easily allow for many user mistakes or quickly lose engagement. As mentioned in the article, Best Practices for Form Design, there are several practices that designers should follow to refrain from these issues.

With the digitization of society, it is normal for people to fill out important documentation online, which usually requires users to set aside a lengthy amount of time. In my personal experience, I found three particular practices that need to be universally followed but are still rare in many websites. Them being:


  1. Multiple Steps:
    this is where a website requires a lengthy amount of field inputs and divides it up by “steps”. In which the user is provided with a progress visual of how much of the document has been completed and how much is left. I find it would be even more helpful to let the user know the approximate amount of time the form takes to fill.

  2. Form Feedback:
    across many websites, if a certain field is completed incorrectly, it’s common for the website to provide an error feedback message without specifying where the error occurred. This makes many users lose patience, which can be easily avoided.

Overlay Considerations for Users

In the article Overuse of Overlays: How to Avoid Misusing Lightboxes, the author discusses the common pitfalls of the excessive use of lightboxes that have abundantly appeared in web design recently. Although overlays can be an effective tool for web designers to display important information that can be overlooked, it can easily lead to user frustration and decrease website performance. To avoid misuse of overlays, the author recommends to provide clear exit options, ensure visual distinction, and conduct proper user testing.

Similarly, in the article Best Practices for Modals / Overlays / Dialog Windows, the author provides more guidelines on how to create overlays that are efficient and empathetic. This is through the use of appropriate modal sizing and placement, user-initiated activation, responsive consideration for mobile screens, and keyboard accessible.
